# Arrays 数组

这一节代码：

```go
package main

import "fmt"

func main() {
	var a [2]string
	a[0] = "Hello"
	a[1] = "World"
	fmt.Println(a[0], a[1])
	fmt.Println(a)
	primes := [6]int{2, 3, 5, 7, 11, 13}
	fmt.Println(primes)
}
```

运行结果：

```text
Hello World
[Hello World]
[2 3 5 7 11 13]
```

## 数组是什么

数组是一组“长度固定、元素类型相同”的连续数据。

Go 数组类型包含两个信息：

1. 元素类型。
2. 数组长度。

比如：

```go
var a [2]string
```

这里 `a` 的类型是：

```go
[2]string
```

意思是：长度为 `2` 的字符串数组。

再比如：

```go
var nums [3]int
```

这里 `nums` 的类型是：

```go
[3]int
```

意思是：长度为 `3` 的整数数组。

## 数组长度是类型的一部分

这是 Go 数组非常重要的点：

```go
[2]int
```

和：

```go
[3]int
```

是两个不同的类型。

示例：

```go
var a [2]int
var b [3]int

a = b // 错误
```

即使它们的元素类型都是 `int`，但长度不同，所以类型不同。

这和 C/C++ 的数组有相似之处，但 Go 把数组长度明确放进了类型系统里。

## 声明数组

声明一个数组：

```go
var a [2]string
```

这会创建一个长度为 `2` 的字符串数组。

数组里的每个元素会自动变成该类型的零值。

因为 `string` 的零值是空字符串 `""`，所以刚声明时：

```go
var a [2]string
fmt.Println(a)
```

输出：

```text
[ ]
```

看起来像空白，是因为两个元素都是空字符串。

如果是 `int` 数组：

```go
var nums [3]int
fmt.Println(nums)
```

输出：

```text
[0 0 0]
```

## 访问数组元素

数组通过下标访问元素。

下标从 `0` 开始。

```go
a[0] = "Hello"
a[1] = "World"
```

这里：

```text
a[0] 是第 1 个元素
a[1] 是第 2 个元素
```

读取：

```go
fmt.Println(a[0], a[1])
```

输出：

```text
Hello World
```

## 数组越界

数组长度是固定的。

如果数组长度是 `2`：

```go
var a [2]string
```

合法下标只有：

```text
0
1
```

访问 `a[2]` 就是越界。

如果编译器能在编译期确定越界，会直接编译报错：

```go
fmt.Println(a[2]) // 编译错误
```

如果下标来自变量，可能会在运行时报错：

```go
i := 2
fmt.Println(a[i]) // 运行时 panic
```

## 数组字面量

可以在创建数组时直接写入元素：

```go
primes := [6]int{2, 3, 5, 7, 11, 13}
```

这里创建了一个长度为 `6` 的 `int` 数组。

它的类型是：

```go
[6]int
```

每个元素分别是：

```text
primes[0] = 2
primes[1] = 3
primes[2] = 5
primes[3] = 7
primes[4] = 11
primes[5] = 13
```

打印：

```go
fmt.Println(primes)
```

输出：

```text
[2 3 5 7 11 13]
```

## 让编译器自动计算长度

如果不想自己数长度，可以写：

```go
primes := [...]int{2, 3, 5, 7, 11, 13}
```

`...` 表示让编译器根据元素个数推断数组长度。

这时 `primes` 的类型仍然是：

```go
[6]int
```

注意：这里得到的是数组，不是切片。

## 指定下标初始化

数组字面量可以指定下标：

```go
a := [5]int{0: 10, 3: 30}
fmt.Println(a)
```

输出：

```text
[10 0 0 30 0]
```

没有指定的元素使用零值。

这种写法适合初始化稀疏数组或强调某些固定位置的含义。

## 部分初始化

如果元素没有写满，剩下的会使用零值。

```go
a := [5]int{1, 2}
fmt.Println(a)
```

输出：

```text
[1 2 0 0 0]
```

## 数组的 len

可以使用 `len` 获取数组长度：

```go
a := [3]int{10, 20, 30}
fmt.Println(len(a))
```

输出：

```text
3
```

数组长度固定，所以 `len(a)` 永远是这个数组类型里的长度。

## 遍历数组

### 1. 普通 for 循环

```go
a := [3]int{10, 20, 30}

for i := 0; i < len(a); i++ {
	fmt.Println(i, a[i])
}
```

输出：

```text
0 10
1 20
2 30
```

### 2. range 遍历

```go
a := [3]int{10, 20, 30}

for i, v := range a {
	fmt.Println(i, v)
}
```

`i` 是下标，`v` 是对应元素的值。

输出：

```text
0 10
1 20
2 30
```

如果只需要值，可以忽略下标：

```go
for _, v := range a {
	fmt.Println(v)
}
```

如果只需要下标：

```go
for i := range a {
	fmt.Println(i)
}
```

## 数组是值类型

Go 数组是值类型。

数组赋值会复制整个数组。

```go
a := [3]int{1, 2, 3}
b := a

b[0] = 100

fmt.Println(a)
fmt.Println(b)
```

输出：

```text
[1 2 3]
[100 2 3]
```

`b := a` 会复制一整份数组。

修改 `b[0]` 不会影响 `a[0]`。

这点和 Go 的切片非常不一样。切片赋值通常共享底层数组。

## 数组作为函数参数

因为数组是值类型，所以把数组传给函数时，也会复制整个数组。

```go
func change(a [3]int) {
	a[0] = 100
}

func main() {
	nums := [3]int{1, 2, 3}
	change(nums)
	fmt.Println(nums)
}
```

输出：

```text
[1 2 3]
```

函数里的 `a` 是外面 `nums` 的副本。

如果想修改原数组，可以传数组指针：

```go
func change(a *[3]int) {
	a[0] = 100
}

func main() {
	nums := [3]int{1, 2, 3}
	change(&nums)
	fmt.Println(nums)
}
```

输出：

```text
[100 2 3]
```

但在实际 Go 开发里，更常用的是传切片，而不是传数组指针。

## 数组和切片的关系

数组是底层数据容器。

切片是对数组某一段的描述。

比如：

```go
primes := [6]int{2, 3, 5, 7, 11, 13}
s := primes[1:4]
```

`primes` 是数组。

`s` 是切片。

`s` 引用了 `primes` 的一部分：

```text
primes: [2 3 5 7 11 13]
          0 1 2 3 4  5

s := primes[1:4]

s:        [3 5 7]
           1 2 3
```

切片不存储完整数据，它只是指向底层数组的一段。

## 什么时候使用数组

Go 日常开发中，数组用得没有切片多。

数组适合这些场景：

1. 长度固定，而且长度本身有明确含义。
2. 小型固定数据，比如 `[2]int` 表示二维坐标。
3. 协议、哈希、加密等固定长度数据，比如 `[16]byte`、`[32]byte`。
4. 需要让长度成为类型的一部分。

示例：

```go
var md5 [16]byte
var sha256 [32]byte
```

这里长度不同就代表不同类型，反而是一种安全约束。

## 什么时候不要使用数组

如果数据长度会变化，或者你只是想保存一组元素，通常应该使用切片。

不推荐：

```go
func printNames(names [3]string) {
	for _, name := range names {
		fmt.Println(name)
	}
}
```

这个函数只能接收 `[3]string`。

如果以后有 2 个名字或 4 个名字，就不能直接传。

更推荐：

```go
func printNames(names []string) {
	for _, name := range names {
		fmt.Println(name)
	}
}
```

切片更灵活。

## 数组和 C++ 数组的对比

你有 C++ 基础，可以这样看：

相同点：

1. 都是一组相同类型的元素。
2. 都可以通过下标访问。
3. 都有固定长度。

重要差异：

1. Go 数组长度是类型的一部分，`[2]int` 和 `[3]int` 是不同类型。
2. Go 数组赋值会复制整个数组。
3. Go 数组传参会复制整个数组。
4. Go 数组不会像 C/C++ 数组那样在很多场景自动退化成指针。
5. Go 更常用切片来表示动态序列。

## 常见坑

### 1. 误以为数组传参会修改原数组

```go
func change(a [3]int) {
	a[0] = 100
}
```

这里改的是副本，不是原数组。

### 2. 误以为 `[3]int` 和 `[4]int` 是同一类数组

它们不是同一个类型。

```go
var a [3]int
var b [4]int

a = b // 错误
```

### 3. 误把数组当成日常动态列表

如果你需要不断追加元素，应该用切片：

```go
s := []int{}
s = append(s, 1)
```

数组长度固定，不能 `append`。

## 总结

数组的核心知识点：

1. 数组长度固定。
2. 数组长度是类型的一部分。
3. 数组元素类型必须相同。
4. 下标从 `0` 开始。
5. 数组默认按元素类型的零值初始化。
6. 数组是值类型，赋值和传参都会复制。
7. 实际开发中，更常用切片；数组更多用于固定长度数据。

一句话记忆：

> 数组是固定长度的一整块值；长度写进类型里，赋值和传参都会复制。
