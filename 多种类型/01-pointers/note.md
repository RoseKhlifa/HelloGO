# Pointers 指针

这一节代码来自 Go 指针的基础示例：

```go
package main

import "fmt"

func main() {
	i, j := 42, 2701

	p := &i
	fmt.Println(*p)
	*p = 21
	fmt.Println(i)

	p = &j
	*p = *p / 37
	fmt.Println(j)
}
```

运行结果：

```text
42
21
73
```

## 指针是什么

指针保存的是一个变量的内存地址。

在 Go 里：

```go
p := &i
```

意思是：取变量 `i` 的地址，并把这个地址保存到变量 `p` 中。

这里：

```go
i := 42
p := &i
```

可以理解为：

```text
i 里面保存的是 42
p 里面保存的是 i 的地址
```

## 两个重要符号

### 1. `&` 取地址

```go
p := &i
```

`&i` 表示获取变量 `i` 的地址。

如果 `i` 的类型是 `int`，那么 `&i` 的类型就是 `*int`。

也就是说：

```go
i := 42
p := &i
```

这里 `p` 的类型是：

```go
*int
```

读作：指向 `int` 的指针。

### 2. `*` 解引用

```go
fmt.Println(*p)
```

`*p` 表示通过指针 `p` 找到它指向的变量，再取出那个变量的值。

如果：

```go
i := 42
p := &i
```

那么：

```go
fmt.Println(*p)
```

打印的就是 `i` 的值：

```text
42
```

`*p` 不只是可以读取，也可以修改。

```go
*p = 21
```

意思是：把 `p` 指向的那个变量改成 `21`。

因为 `p` 指向的是 `i`，所以这句代码等价于：

```go
i = 21
```

## 示例逐步讲解

### 第 1 步：定义两个变量

```go
i, j := 42, 2701
```

这行代码定义了两个 `int` 变量：

```text
i = 42
j = 2701
```

### 第 2 步：让 p 指向 i

```go
p := &i
```

`&i` 表示 `i` 的地址。

所以 `p` 现在指向 `i`。

可以想成：

```text
p -> i -> 42
```

### 第 3 步：通过 p 读取 i

```go
fmt.Println(*p)
```

`*p` 表示读取 `p` 指向的变量。

因为 `p` 指向 `i`，所以 `*p` 就是 `i` 的值。

输出：

```text
42
```

### 第 4 步：通过 p 修改 i

```go
*p = 21
```

这行不是修改 `p` 本身，而是修改 `p` 指向的变量。

因为 `p` 指向 `i`，所以 `i` 被改成了 `21`。

```go
fmt.Println(i)
```

输出：

```text
21
```

### 第 5 步：让 p 改为指向 j

```go
p = &j
```

现在 `p` 不再指向 `i`，而是指向 `j`。

可以想成：

```text
p -> j -> 2701
```

注意：这一步只是改变了 `p` 保存的地址，不会改变 `i` 的值。

### 第 6 步：通过 p 修改 j

```go
*p = *p / 37
```

因为 `p` 现在指向 `j`，所以：

```go
*p
```

就是 `j` 的值，也就是 `2701`。

这行代码等价于：

```go
j = j / 37
```

计算结果：

```text
2701 / 37 = 73
```

所以：

```go
fmt.Println(j)
```

输出：

```text
73
```

## Go 指针和 C++ 指针的相同点

你有 C++ 基础，可以这样类比：

```go
i := 42
p := &i
fmt.Println(*p)
*p = 21
```

和 C++ 中的思路很像：

```cpp
int i = 42;
int* p = &i;
cout << *p << endl;
*p = 21;
```

共同点：

1. `&` 都表示取地址。
2. `*` 都表示通过指针访问指向的值。
3. 可以通过指针修改原变量。
4. 指针变量本身保存的是地址。

## Go 指针和 C++ 指针的重要区别

### 1. Go 没有指针运算

Go 的指针不能做加减运算。

C++ 中可能会写：

```cpp
p++;
p = p + 1;
```

Go 中不能这样写：

```go
p++      // 错误
p = p+1  // 错误
```

这是 Go 指针最重要的安全限制之一。

Go 允许你使用指针，但不允许你随意移动地址，这样可以减少很多内存错误。

### 2. Go 不需要手动释放内存

C++ 中经常会考虑：

```cpp
new
delete
```

Go 有垃圾回收机制，普通 Go 代码里不需要手动 `delete`。

Go 也有 `new`：

```go
p := new(int)
*p = 42
```

但 `new(int)` 只是分配一个 `int` 并返回 `*int`，后续释放由 Go 运行时管理。

### 3. Go 没有 `->`

C++ 里访问结构体指针成员常写：

```cpp
p->Name
```

Go 里没有 `->`，统一使用 `.`。

```go
type User struct {
	Name string
}

u := User{Name: "Tom"}
p := &u

fmt.Println(p.Name)
```

这里 `p.Name` 实际上等价于：

```go
(*p).Name
```

Go 会自动帮你做结构体指针的解引用。

### 4. Go 函数参数默认仍然是值传递

Go 里面函数传参默认是值传递。

```go
func change(x int) {
	x = 100
}

func main() {
	i := 42
	change(i)
	fmt.Println(i)
}
```

输出还是：

```text
42
```

如果想在函数里修改外面的变量，需要传指针：

```go
func change(p *int) {
	*p = 100
}

func main() {
	i := 42
	change(&i)
	fmt.Println(i)
}
```

输出：

```text
100
```

注意：传指针本身也是值传递，只是复制的是地址。函数里面拿到的是同一个变量的地址，所以可以修改原变量。

### 5. Go 指针可以是 nil

指针的零值是 `nil`。

```go
var p *int
fmt.Println(p)
```

此时 `p` 没有指向任何有效变量。

不能直接解引用 `nil` 指针：

```go
var p *int
fmt.Println(*p) // 运行时 panic
```

使用指针前，尤其是函数返回的指针、结构体字段里的指针，要注意是否为 `nil`。

## 指针常见用途

### 1. 在函数中修改外部变量

```go
func reset(x *int) {
	*x = 0
}

func main() {
	n := 10
	reset(&n)
	fmt.Println(n)
}
```

输出：

```text
0
```

适合：函数需要修改调用者传进来的变量。

### 2. 避免复制较大的结构体

如果结构体很大，每次传值都会复制一份。

可以传指针：

```go
type User struct {
	Name string
	Age  int
}

func updateAge(u *User) {
	u.Age++
}
```

这样函数接收的是地址，不需要复制整个结构体。

### 3. 表示“可选值”

普通 `int` 的零值是 `0`，但有时需要区分：

1. 用户真的传了 `0`。
2. 用户没有传值。

这时可以用指针：

```go
var age *int
```

如果 `age == nil`，表示没有值。

如果 `age != nil`，表示有值，可以通过 `*age` 读取。

### 4. 方法需要修改接收者

Go 的方法可以有值接收者，也可以有指针接收者。

```go
type Counter struct {
	N int
}

func (c *Counter) Inc() {
	c.N++
}
```

这里必须用 `*Counter`，因为 `Inc` 要修改原来的 `Counter`。

如果写成：

```go
func (c Counter) Inc() {
	c.N++
}
```

修改的是副本，不会影响原来的值。

## 使用指针时的注意事项

### 1. 分清楚修改指针还是修改指针指向的值

```go
p = &j
```

这句是修改 `p`，让它指向别的变量。

```go
*p = 21
```

这句是修改 `p` 指向的变量。

这是本节示例最重要的区别。

### 2. 不要解引用 nil 指针

```go
var p *int
fmt.Println(*p)
```

这会导致运行时错误。

应该先判断：

```go
if p != nil {
	fmt.Println(*p)
}
```

### 3. 指针不是“引用类型”的唯一形式

Go 里还有一些类型本身就包含对底层数据的引用，例如：

1. slice
2. map
3. channel
4. function
5. interface

比如 slice 传给函数时，虽然 slice 头是值传递，但它指向的底层数组可以被修改。

所以不是所有“函数里能改外面数据”的情况都必须显式写指针。

### 4. 返回局部变量地址是允许的

在 C++ 中，返回局部变量地址通常是严重错误。

但 Go 里可以这样写：

```go
func makeInt() *int {
	x := 42
	return &x
}
```

Go 编译器会做逃逸分析。如果发现 `x` 在函数结束后还要继续使用，就会把它安排到合适的位置，由垃圾回收管理生命周期。

所以 Go 中“返回局部变量地址”本身不是错误。

### 5. 不要为了“看起来高效”滥用指针

小的基础类型，比如 `int`、`bool`、`float64`，直接传值通常更简单。

只有在这些情况才更适合用指针：

1. 需要修改原值。
2. 结构体较大，想避免复制。
3. 需要表达 `nil`，也就是“没有值”。
4. 方法需要修改接收者。

## 这节示例的核心总结

这节代码主要讲了三件事：

1. `&i` 可以取得变量 `i` 的地址。
2. `*p` 可以访问 `p` 指向的变量。
3. 修改 `*p` 会影响被指向的原变量。

一句话记忆：

> `&` 是取地址，`*` 是沿着地址找到值；改 `p` 是换地址，改 `*p` 是改地址里的值。
